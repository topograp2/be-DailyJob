### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상:Accept: 다음으로 오는(ex.image/avif) contentType을 추출하여 응답으로 보내주었더니 svg 등의 타입이 지원되지 않는 문제가 있었다.
- 처음 추측한 원인: 제대로 split하지 않는다고 생각했다.
- 원인을 확인하기 위해 시도한 방법: 로그를 찍어서 현재 split되고 있는 contentType을 직접 확인했다.
- 실제 원인: 실제로는 split 자체는 잘 되고 있었으나 svg -> image/svg+xml과 같이 각각 컨텐츠 타입이 존재하기때문에 Accept: 다음으로 오는 타입으로 무조건 지정하면 안되었다.
- 해결 방법: 각 파일의 확장자명, contentType을 Map에 담아둔 다음에 header에서 확장자명을 뽑아내서 types.get을 통해 실제 contentType을 가져옴
```
 private static final Map<String, String> types = new HashMap<>();
    static {
        types.put("html", "text/html");
        types.put("css", "text/css");
        types.put("svg", "image/svg+xml");
        types.put("js", "application/javascript");
        types.put("jpg", "image/jpeg");
        types.put("png", "image/png");
        types.put("ico", "image/x-icon");
    }
```
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 어떤 확장자명에 따라 어떤 컨텐츠타입으로 보내주어야 할지를 미리 확인하기
